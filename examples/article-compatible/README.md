# Примеры CacheSim, совместимые со статьёй

Эти примеры повторяют идеи паттернов доступа из статьи, но приведены к ограниченному подмножеству C, которое реально поддерживает `cats`.

Формат запуска:

```bash
cd worker-cache-interpreter/examples/article-compatible
/home/ivan/work/gybryd-analytic/worker-cache-interpreter/cats <имя_файла> json
```

Рабочие примеры:

- `u1.c` — линейный `unit_stride` доступ.
- `s2.c` — `constant_stride` с шагом `2`.
- `br.c` — `broadcast`, повторное чтение одного элемента.

Ограничения, подтверждённые локально:

- Нельзя использовать `for (int i = ...)`; счётчик надо объявлять до цикла.
- Для тестов воркера нельзя использовать массивы `double` и `float`.
- Лучше держать короткие имена файлов, потому что интерпретатор обрезает имя result-файла.
- Примеры с косвенной индексацией вроде `a[i] = b[idx[i]]` пока небезопасны для тестов: локально они приводили к `stack smashing detected`.

Ожидаемые result-файлы после успешного запуска:

- `u1_result`
- `s2_result`
- `br_result`
